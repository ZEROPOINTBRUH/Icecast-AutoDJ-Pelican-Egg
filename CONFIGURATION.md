# 🎵 AutoDJ-Extreme Configuration System

## Overview

The AutoDJ-Extreme Extreme Edition v2.0 includes a **centralized environment-based configuration system** that ensures consistency, security, and ease of deployment across all components.

## ⚡ Quick Start

### 1. Create Your Configuration

```bash
# Copy the example configuration
cp .env.example .env

# Edit with your values
nano .env  # Linux/Mac
notepad .env  # Windows
```

### 2. Generate All Configurations

```bash
# Make the script executable
chmod +x scripts/generate-configs.sh

# Generate configurations from .env
./scripts/generate-configs.sh
```

### 3. Start Services

```bash
# Docker
docker-compose up -d

# Or systemd
sudo systemctl restart icecast2
```

### 4. Verify

```bash
# Check stream status
curl http://localhost:8000/status.xsl

# Test public stream
ffplay http://localhost:8000/autodj.mp3
```

## 📋 Configuration Files

### Generated by `generate-configs.sh`

| File | Purpose | Source |
|------|---------|--------|
| `configs/icecast.xml` | Icecast server configuration | `.env` variables |
| `configs/radio.liq` | Liquidsoap audio engine config | `.env` variables |
| `configs/ezstream.v0.xml` | Ezstream v0.x fallback config | `.env` variables |
| `configs/ezstream.v1.xml` | Ezstream v1.x fallback config | `.env` variables |

### Configuration Reference Files

| File | Purpose | Usage |
|------|---------|-------|
| `.env.defaults` | Default/fallback values | Automatically loaded by script |
| `.env.example` | Configuration template | Copy to `.env` and customize |
| `ENVIRONMENT_CONFIG.md` | Complete variable reference | Documentation |

### Do Not Edit Manually

These files are **auto-generated from `.env`**:
- ❌ Don't edit `configs/icecast.xml` directly
- ❌ Don't edit `configs/radio.liq` directly  
- ❌ Don't edit `configs/ezstream*.xml` directly

Instead, **edit `.env` and run `generate-configs.sh`**

## 🔐 Security Best Practices

### Critical: Change Default Passwords

```bash
# BEFORE DEPLOYMENT: Edit .env with strong passwords
ADMIN_PASSWORD=YourSecure16CharPassword123!
SOURCE_PASSWORD=YourSecure16CharPassword456!
RELAY_PASSWORD=YourSecure16CharPassword789!
AUTODJ_PASSWORD=YourSecure16CharPassword012!
LIVE_PASSWORD=YourSecure16CharPassword345!
STREAM_PASSWORD=YourSecure16CharPassword678!
```

### Password Requirements

- ✅ Minimum 12 characters (recommend 16+)
- ✅ Mix of uppercase and lowercase letters
- ✅ Numbers and special symbols
- ✅ Different password for each service
- ✅ No common words or patterns

### Generate Strong Passwords

```bash
# Linux/Mac
openssl rand -base64 16

# Windows PowerShell
-join ((65..90) + (97..122) + (48..57) | Get-Random -Count 16 | % {[char]$_})

# Online Generator
https://www.lastpass.com/features/password-generator
```

### Protect Your .env File

```bash
# Restrict permissions
chmod 600 .env

# Keep out of version control
echo ".env" >> .gitignore
git rm --cached .env
```

## 🎯 Common Use Cases

### Public Radio Station

```bash
# Allow public listening, no password required
AUTODJ_PUBLIC=1          # Public AutoDJ stream
LIVE_PUBLIC=1            # Public live input
STREAM_PUBLIC=1          # Public relay stream

# Typical setup
ADMIN_PASSWORD=YourAdminPass123!
SOURCE_PASSWORD=YourSourcePass456!
```

### Private/Corporate Stream

```bash
# All streams private, authentication required
AUTODJ_PUBLIC=0          # Private AutoDJ
LIVE_PUBLIC=0            # Private live input
STREAM_PUBLIC=0          # Private relay

# Use corresponding passwords
AUTODJ_PASSWORD=PrivatePass789!
LIVE_PASSWORD=PrivatePass012!
STREAM_PASSWORD=PrivatePass345!
```

### High-Bitrate Studio

```bash
# Professional quality streaming
MP3_BITRATE=192         # Higher bitrate
OGG_BITRATE=192         # Higher bitrate
ICECAST_MAX_CLIENTS=500 # Adjust for bandwidth

# Example: 192kbps × 500 clients = 96 Mbps bandwidth needed
```

### Low-Bandwidth Mobile

```bash
# Reduced quality for limited bandwidth
MP3_BITRATE=96          # Lower bitrate
OGG_BITRATE=96          # Lower bitrate
ICECAST_MAX_CLIENTS=2000 # Can support more listeners
```

## 📊 Environment Variables Reference

### Security Credentials

```bash
ADMIN_USER=admin                    # Admin username
ADMIN_PASSWORD=password123          # Admin panel password
SOURCE_PASSWORD=sourcepass123       # Stream input password
RELAY_PASSWORD=relaypass123         # Relay server password
AUTODJ_PASSWORD=autodjpass123       # AutoDJ mount password
LIVE_PASSWORD=livepass123           # Live mount password
STREAM_PASSWORD=streampass123       # Stream mount password
```

### Network Configuration

```bash
ICECAST_HOST=localhost              # Icecast hostname
ICECAST_PORT=8000                   # Icecast port
ICECAST_MAX_CLIENTS=1000            # Max listeners
ICECAST_BURST_SIZE=65535            # Connection burst size

WEB_HOST=0.0.0.0                    # Web UI bind address
WEB_PORT=8080                       # Web UI port
```

### Stream Settings

```bash
AUTODJ_PUBLIC=1                     # 1=public, 0=private
LIVE_PUBLIC=0                       # 1=public, 0=private
STREAM_PUBLIC=1                     # 1=public, 0=private

MP3_BITRATE=128                     # MP3 quality (kbps)
OGG_BITRATE=128                     # OGG quality (kbps)
```

### Content Configuration

```bash
MUSIC_DIR=/home/container/music     # Music files location
PLAYLIST_DIR=/home/container/playlists
ADS_DIR=/home/container/ads
JINGLES_DIR=/home/container/jingles
LOG_DIR=/home/container/log

ENABLE_ADS=false                    # Enable ad rotation
AD_FREQUENCY=5                      # Ad every N tracks
ENABLE_JINGLES=false                # Enable jingles
JINGLE_FREQUENCY=3                  # Jingle every N tracks
ENABLE_LIQUIDSOAP=true              # Use Liquidsoap
```

### Logging & Monitoring

```bash
LOG_LEVEL=3                         # 0=critical, 4=debug
MONITOR_INTERVAL=30                 # Health check frequency (seconds)
```

### Station Metadata

```bash
STATION_NAME=AutoDJ-Extreme
STATION_DESCRIPTION=Professional Radio
STATION_LOCATION=Earth
ADMIN_EMAIL=admin@example.com
```

## 🐳 Docker Integration

### Using with Docker Compose

```yaml
version: '3.8'
services:
  autodj:
    image: zeropointbruh/autodj-extreme:latest
    container_name: autodj-extreme
    
    ports:
      - "${ICECAST_PORT}:8000"
      - "${WEB_PORT}:8080"
    
    volumes:
      - ./music:/home/container/music
      - ./logs:/home/container/log
      - ./configs:/mnt/configs
    
    environment:
      - ADMIN_USER=${ADMIN_USER}
      - ADMIN_PASSWORD=${ADMIN_PASSWORD}
      - SOURCE_PASSWORD=${SOURCE_PASSWORD}
      - ICECAST_PORT=${ICECAST_PORT}
      - MP3_BITRATE=${MP3_BITRATE}
      # ... add all vars or use env_file
    
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/status.xsl"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 40s
```

Or use `env_file` for cleaner approach:

```yaml
services:
  autodj:
    image: zeropointbruh/autodj-extreme:latest
    ports:
      - "8000:8000"
      - "8080:8080"
    volumes:
      - ./music:/home/container/music
      - ./logs:/home/container/log
    env_file: .env           # ← Load all variables from .env
    restart: unless-stopped
```

### Using with Docker CLI

```bash
docker run -d \
  --name autodj-extreme \
  -p 8000:8000 \
  -p 8080:8080 \
  --env-file .env \
  -v ./music:/home/container/music \
  -v ./logs:/home/container/log \
  zeropointbruh/autodj-extreme:latest
```

## 🐛 Troubleshooting

### "Authentication Failed" After Update

**Cause:** Password changed in `.env` but configs not regenerated

**Solution:**
```bash
./scripts/generate-configs.sh
docker restart autodj-extreme
```

### Mount Points Not Accessible

**Cause:** Mount names don't match between Icecast and Liquidsoap

**Solution:**
- Check `.env` - ensure single definition
- Regenerate configs: `./scripts/generate-configs.sh`
- Verify `configs/icecast.xml` has matching mount names

### Listeners Can't Connect

**Cause:** `AUTODJ_PUBLIC=0` requires password

**Solution:**
```bash
# For public access:
AUTODJ_PUBLIC=1

# Then regenerate:
./scripts/generate-configs.sh
docker restart autodj-extreme
```

### Different Settings on Each Restart

**Cause:** Environment variables override `.env`

**Solution:**
- Unset environment variables: `unset ADMIN_PASSWORD`
- Or explicitly load `.env`: `source .env && ./scripts/generate-configs.sh`

## 📝 Variable Scope

| Variable | Used By | Purpose |
|----------|---------|---------|
| `ADMIN_*` | Icecast, Web UI | Admin panel access |
| `*_PASSWORD` | Icecast, Liquidsoap | Authentication |
| `ICECAST_*` | Icecast, Liquidsoap | Server configuration |
| `*_PUBLIC` | Icecast, Web UI | Access control |
| `*_BITRATE` | Liquidsoap | Stream quality |
| `*_DIR` | Liquidsoap, Scripts | File locations |
| `LOG_*` | Icecast, Liquidsoap | Logging |
| `STATION_*` | Icecast, Web UI | Metadata |

## 🚀 Advanced: Custom Configurations

If you need advanced configurations not covered by environment variables:

1. **Modify the template** in `scripts/generate-configs.sh`
2. **Add new variables** to `.env.defaults` and `.env.example`
3. **Regenerate**: `./scripts/generate-configs.sh`
4. **Test thoroughly** before production deployment
